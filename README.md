# my-env-template

研究・開発・実務で再現性の高い計算環境（C/C++ / Julia / Python / 
CUDA）をDockerベースで統一的に構築するためのテンプレート集です。

本リポジトリは以下の2系統を扱います：

-   **research/** --- 大学・個人研究向け\
-   **work/** --- 仕事向け（予定）

環境の違いや制約を吸収しつつ、長期的にメンテ可能な**バージョン別フォルダ構成**を採用しています。

------------------------------------------------------------------------

## 📁 Folder Structure

    my-env-template/
    ├── research/
    │   └── v1.0.0/
    │       ├── docker/
    │       ├── README.md
    │       └── CHANGELOG.md
    │

    │   └── （今後 v1.0.0/ を追加予定）
    │
    └── README.md

## 🎯 Purpose

-   利用環境ごとの差異（GPU/CPU、依存ライブラリ、ポリシー）をテンプレ化
-   完全再現性のある計算環境をいつでも復元
-   バージョンごとに依存関係を固定し、プロジェクトの再現性確保

------------------------------------------------------------------------

## 🚀 How to Use

### 1. 用途を選択する

    research/  または  work/

### 2. バージョンを選択する

例：

    research/v1.0.0/
    work/v1.0.0/

### 3. 各バージョンの README に従って環境構築

Dockerfile / docker-compose などのセットアップ方法は\
各バージョンの README を参照。

## 🧩 Versioning Policy（Semantic Versioning）

本リポジトリは **MAJOR.MINOR.PATCH** 形式でバージョン管理します。

- **PATCH**（例：v1.0.0 → v1.0.1）  
  小さな修正・依存更新  
  → *同じフォルダ内で変更し、CHANGELOG.md に追記*

- **MINOR**（例：v1.0.0 → v1.1.0）  
  互換性を保った機能追加  
  → *フォルダはそのまま、CHANGELOG.md を更新*

- **MAJOR**（例：v1.0.0 → v2.0.0）  
  互換性のない大きな変更（OS/CUDA更新など）  
  → *新しいフォルダ（例：v2.0.0/）を作成*

**原則：過去バージョンはそのまま保持し、再現性を最優先する。**
